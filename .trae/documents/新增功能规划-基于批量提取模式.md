## 一、批量提取功能的技术模式分析

### 核心特征
1. **单文件输入** → **多文件输出**（ZIP 打包）
2. **可视化页面选择**（缩略图网格 + 复选框）
3. **批量操作**（全选/反选/范围选择）
4. **技术栈**：pdf-lib（页面操作）+ pdf.js（缩略图渲染）+ JSZip（打包下载）

### 成功要素
- 用户痛点明确：需要批量处理页面，但不想手动操作多次
- 交互直观：可视化选择，所见即所得
- 技术难度低：纯 API 组合，无复杂算法

#### 1. 批量旋转 ⭐⭐
**功能描述：**
- 上传一个 PDF
- 显示所有页面缩略图网格
- 勾选需要旋转的页面（支持全选/反选/范围选择）
- 选择旋转角度（90°/180°/270°）
- 导出为新 PDF

**用户场景：**
- 扫描文档时部分页面方向错误
- 混合横向/纵向页面需要统一方向

**技术实现：**
```javascript
// 核心逻辑（约 50 行）
const pdfDoc = await PDFDocument.load(bytes);
const pages = pdfDoc.getPages();
selectedPages.forEach(index => {
  pages[index].rotate(degrees(rotationAngle));
});
const outputBytes = await pdfDoc.save();
```

**UI 设计：**
- 复用批量提取的页面网格组件
- 每个缩略图下方显示旋转角度选择器
- 顶部工具栏：旋转角度选择（90°/180°/270°）+ 全选/反选按钮

**预计代码量：** 200-250 行（复用 batch-extract.js 结构）

#### 4. 批量添加页码 ⭐⭐
**功能描述：**
- 上传一个 PDF
- 设置页码格式（第 X 页/共 Y 页、X/Y、纯数字）
- 选择页码位置（左下角/中下/右下角）
- 设置字体大小、颜色
- 可选择起始页码（如从第 3 页开始编号）
- 导出为新 PDF

**用户场景：**
- 为扫描文档添加页码
- 合并多个 PDF 后统一编号
- 打印前添加页码

**技术实现：**
```javascript
// 核心逻辑（约 80 行）
const pdfDoc = await PDFDocument.load(bytes);
const pages = pdfDoc.getPages();
const totalPages = pages.length;
pages.forEach((page, index) => {
  if (index >= startPage - 1) {
    const pageNum = index - startPage + 1;
    const text = formatPageNumber(pageNum, totalPages, format);
    page.drawText(text, { x, y, size, font, color });
  }
});
const outputBytes = await pdfDoc.save();
```

**UI 设计：**
- 不需要页面网格，直接显示配置表单
- 页码格式选择器（下拉菜单）
- 位置选择（九宫格点击选择）
- 预览区域显示页码效果

**预计代码量：** 250-350 行

#### 6. PDF 分割为等份 ⭐⭐
**功能描述：**
- 上传一个 PDF
- 选择分割方式：
  - 按页数均分（如 20 页分成 4 个文件，每个 5 页）
  - 按文件数量均分（如分成 3 个文件）
  - 自定义每份页数（如 5,10,5）
- 打包成 ZIP 下载

**用户场景：**
- 将大文档拆分为小章节
- 按固定页数分割便于打印
- 分发文档的不同部分

**技术实现：**
```javascript
// 核心逻辑（约 70 行）
const srcDoc = await PDFDocument.load(bytes);
const totalPages = srcDoc.getPageCount();
const chunks = splitIntoChunks(totalPages, mode, params);
chunks.forEach((range, index) => {
  const newDoc = await PDFDocument.create();
  const [pages] = await newDoc.copyPages(srcDoc, range);
  pages.forEach(p => newDoc.addPage(p));
  zip.file(`part_${index + 1}.pdf`, await newDoc.save());
});
```

**UI 设计：**
- 分割方式选择器（单选按钮组）
- 根据选择显示不同的参数输入框
- 预览分割结果（如"将生成 4 个文件：1-5, 6-10, 11-15, 16-20"）

**预计代码量：** 250-300 行

**功能描述：**
- 上传主 PDF
- 上传替换 PDF

- 更新文档中的某些页面
- 替换错误的扫描页

```javascript
// 核心逻辑（约 80 行）
const mainDoc = await PDFDocument.load(mainBytes);

selectedPages.forEach((mainIndex, replaceIndex) => {
  mainDoc.removePage(mainIndex);
  const [page] = await mainDoc.copyPages(replaceDoc, [replaceIndex]);
  mainDoc.insertPage(mainIndex, page);
});
const outputBytes = await mainDoc.save();
```

- 两个上传区域（主 PDF + 替换 PDF）
- 主 PDF 显示页面网格供选择

#### 8. 奇偶页分离 ⭐⭐
**功能描述：**
- 上传一个 PDF
- 选择分离方式：
  - 仅保留奇数页
  - 仅保留偶数页

- 导出结果

**用户场景：**
- 双面打印准备（先打奇数页，翻转后打偶数页）
- 提取特定页面类型
- 特殊排版需求

**技术实现：**
```javascript
// 核心逻辑（约 50 行）
const srcDoc = await PDFDocument.load(bytes);
const oddPages = [], evenPages = [];
for (let i = 0; i < totalPages; i++) {
  (i % 2 === 0 ? oddPages : evenPages).push(i);
}
// 根据选择创建新文档
```

**UI 设计：**
- 简单的单选按钮组（奇数/偶数/分离为两个）
- 预览结果（如"奇数页：1,3,5,7... 共 10 页"）

**预计代码量：** 150-200 行

## 七、风险评估

### 技术风险
- **低风险**：所有功能基于 pdf-lib 标准 API
- **中风险**：页面重排的拖拽排序需要测试兼容性
- **低风险**：ZIP 打包已有成熟方案（JSZip）

### 用户体验风险
- 页面网格渲染性能：超过 100 页时可能卡顿
  - **解决方案**：虚拟滚动或分页加载
- 内存限制：大文件 + 多页面缩略图可能超出浏览器限制
  - **解决方案**：懒加载缩略图，限制同时渲染数量
